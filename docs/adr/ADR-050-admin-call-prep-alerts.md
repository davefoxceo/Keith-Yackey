# ADR-050: Admin Call Prep Alerts -- Automated Client Briefings

**Status:** Proposed
**Date:** 2026-03-24
**Related:** ADR-035 (Admin Dashboard), ADR-037 (Belt Progression), ADR-034 (Stripe Payments), ADR-049 (Smart Brain Accountability)

---

## Context

Keith Yackey runs weekly coaching calls with his clients. Before each call, he needs to know where a client stands -- their scores, their recent conversations with the AI, any red flags, and what to bring up. Today, Keith has to manually dig through the admin dashboard and piece together this picture for each client. That prep eats time and risks missing something important. If a client's Partner dial cratered last week and Keith does not bring it up on the call, the client feels like nobody is paying attention.

The system already has all the data: Five Dials scores, leading and lagging scores, grades, conversation history, belt levels, and subscription tiers. Automating the call prep turns that raw data into a focused briefing that arrives before Keith picks up the phone.

---

## Decision

**Build a CallPrepService that generates automated client briefing reports before Keith's scheduled coaching calls, delivered via email digest and an in-admin notification panel.**

### Call Schedule Configuration

Keith sets his call schedule in the admin panel. The schedule defines recurring time slots.

| Field | Type | Example |
|-------|------|---------|
| `day_of_week` | enum | Tuesday, Thursday |
| `time` | HH:mm (local timezone) | 10:00 |
| `client_ids` | string[] | List of user IDs on that call slot |
| `recurrence` | weekly / biweekly | weekly |

Keith can add, edit, or remove call slots from the admin settings page. Each slot can hold one or more clients.

### Prep Report Generation

The system generates a prep report for each client 1 hour before the scheduled call time. The report contains:

#### Client Summary Header

Client name, belt level, subscription tier, member-since date, days active in last 30 days.

#### Five Dials Snapshot

Current score for each dial (Provider, Protector, Partner, Player, Faith) with week-over-week trend arrows and 4-week sparklines.

#### Composite Scores

Leading score and lagging score with 4-week trend lines. Grade (A-F) with a one-sentence explanation of what drove the grade.

#### Recent AI Conversations

Summaries of the last 3 conversations: date, mode (Coach/Mentor/Accountability/Crisis), key topics, approximate duration. Generated by the LearningService, not raw transcripts.

#### Red Flags

Automatically highlighted: dial drops of 2+ points, missed assessments, 5+ days inactive, grade decline over 2 weeks, accountability interventions sent (ADR-049).

#### Suggested Talking Points

Claude generates 2-3 specific talking points from the client's scores, trends, summaries, and red flags. Example: "Ask about the Partner dial drop -- he mentioned communication struggles in his last 2 conversations." The prompt steers Claude to produce actionable items framed in Keith's coaching language.

### Delivery Mechanisms

#### Email Digest

- Sent 1 hour before the first call of the day
- Contains all client prep reports for that day's calls
- HTML email with expandable sections per client
- Red flag clients are listed first with a red badge

#### In-Admin Notification Panel

- "Upcoming Calls" page accessible from the admin sidebar
- Shows today's scheduled calls with countdown timers
- Each call slot expands to show client cards with the full prep report
- Client cards are color-coded by grade: green (A/B), yellow (C), red (D/F)

#### Red Alert Badge

If ANY client on an upcoming call has a grade below C, the admin navigation shows a red alert badge on the "Upcoming Calls" link. The badge count reflects the number of at-risk clients on today's schedule. This ensures Keith never walks into a call blind about a struggling client.

## Implementation

```
src/services/call-prep.service.ts        -- report generation and scheduling
src/services/call-schedule.service.ts    -- CRUD for call schedule configuration
src/models/call-prep-report.model.ts     -- prep report schema (id, call_slot_id, client_id, dials, scores, grade, summaries, red_flags, talking_points)
src/models/call-schedule.model.ts        -- call schedule schema
```

### CallPrepService API

- `generateReport(callSlotId) -> PrepReport[]` -- generates reports for all clients in a call slot
- `getUpcomingReports(adminId) -> PrepReport[]` -- returns prep reports for today's calls
- `regenerateReport(reportId) -> PrepReport` -- manually regenerate a single report with fresh data

### Scheduled Job

A cron job (or manual trigger from admin) runs the prep generation: query call slots within 60 minutes, generate reports for each client, send email digest via the notification service (ADR-041), and store reports in DataStore for the admin panel.

### Admin UI Routes

| Route | View | Purpose |
|-------|------|---------|
| `/admin/calls` | Upcoming Calls | Today's call schedule with client prep cards |
| `/admin/calls/schedule` | Call Schedule | Configure recurring call slots and assign clients |
| `/admin/calls/:slotId` | Call Detail | Full prep reports for a specific call slot |

## Consequences

### Positive

- Keith walks into every call fully prepared without manual data digging
- Red flag prioritization ensures struggling clients get attention first
- AI-generated talking points keep calls focused on what matters most for each client
- Email delivery means Keith can review on his phone before calls even if he is not at his computer
- Regenerate option lets Keith get a fresh report if a client submits data between generation and call time

### Negative

- Claude API calls for talking point generation add cost per client per call (mitigation: cache reports, only regenerate on demand)
- Cron-based scheduling requires reliable job execution (mitigation: manual trigger fallback in admin panel)
- Conversation summaries depend on LearningService quality (mitigation: summaries are supplementary, raw conversation access is still available via ADR-035)

### Risks

- If call schedule is not maintained, reports will not generate (mitigation: admin panel shows "no upcoming calls configured" warning)
- Stale data if client submits assessment between report generation and call time (mitigation: regenerate button, reports show generation timestamp)
