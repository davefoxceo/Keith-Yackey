# ADR-043: Upsell and Product Recommendation Engine

**Status:** Proposed
**Date:** 2026-03-24

## Context

Keith Yackey's business includes multiple revenue streams beyond the Coach Keith AI subscription: a comic book series ($119-299/year), The Married Game book, live events (Man In The Arena Tour), the Ascend Brotherhood mastermind group, and 1-on-1 coaching packages. Currently, there is no in-app mechanism to surface these products to users at relevant moments. Users discover them only if they visit Keith's external website or social media.

The coaching AI is uniquely positioned to make contextual recommendations because it knows the user's dial scores, conversation history topics, engagement patterns, and belt level. A user struggling with their Partner dial who has had three conversations about communication is a natural candidate for a live event where Keith teaches the HEAR framework in person. A highly engaged user at Brown belt is a strong candidate for the Ascend Brotherhood upgrade.

The key constraint is tone. These recommendations must feel like Coach Keith suggesting a next step in the user's growth journey, not like advertisements interrupting a coaching session.

## Decision

Build a **RecommendationService** that evaluates user context and returns up to 3 relevant product suggestions, delivered through Coach Keith's voice and placed at natural transition points in the user experience.

### Product Catalog

| Product | Price | Target User Profile | Priority |
|---------|-------|-------------------|----------|
| **The Married Game (Book)** | $24.99 | New users, free tier, anyone who hasn't purchased | High (low barrier) |
| **Comic Book Subscription** | $119-299/yr | Users who engage with content library, visual learners | Medium |
| **Man In The Arena Tour** | $197-497/event | Users with 2+ weeks active, struggling in specific dial areas | High (time-sensitive) |
| **Ascend Brotherhood** | $2,500+/yr | High engagement, Brown+ belt, 30+ day streak, already seeing results | High (high LTV) |
| **1-on-1 Coaching with Keith** | $5,000+ | Users who hit a plateau, complex situations the AI flags as needing human coaching | Low volume, high value |

### Recommendation Rules

The engine uses deterministic rules (not ML) based on user profile signals:

#### Signal-to-Product Mapping

| Signal | Product Recommendation | Rationale |
|--------|----------------------|-----------|
| Low Partner dial (< 3) + 3 conversations about communication | Man In The Arena Tour + HEAR framework episode | Live events create breakthroughs for communication issues |
| Low Faith dial (< 3) + early in journey (< 2 weeks) | The Married Game book | Book provides foundational framework |
| High engagement (streak > 14 days) + Brown belt or above | Ascend Brotherhood | User is ready for the next level of commitment |
| AI flags conversation as "beyond AI scope" (safety, deep trauma, complex legal) | 1-on-1 Coaching with Keith | Human coaching needed for situations the AI should not handle alone |
| Active content consumer (5+ episodes viewed) + visual learner signals | Comic Book Subscription | Matches their content consumption style |
| Free tier user + completed 3+ assessments | Any paid subscription tier | User is engaged but not yet paying |
| Declining dial scores over 3 weeks despite active engagement | Man In The Arena Tour or 1-on-1 Coaching | User needs more intensive intervention |

#### Recommendation Cooldowns

- Same product: do not recommend again for 14 days after dismissal
- Any product: maximum 1 recommendation per coaching session
- Recently purchased: never recommend a product the user already owns
- First 3 days: no recommendations at all (let the user settle in)

### Placement and Delivery

#### 1. End of Coaching Conversation (Primary)

After a meaningful coaching exchange (5+ messages), Coach Keith can naturally suggest a next step:

> "You're making real progress on the communication front. If you want to go deeper on the HEAR framework, I'm teaching it live at the next Man In The Arena event in Dallas. It's the kind of thing that clicks different in person."

This is generated by appending a recommendation context to the AI prompt when the RecommendationService returns a match. The AI weaves it into Keith's natural coaching voice -- it is never a templated banner or popup.

#### 2. Content Library "Recommended For You" Section

A dedicated section at the top of the content library showing 1-3 items:
- Mix of free content (episodes, resources) and product recommendations
- Clearly labeled: "Coach Keith recommends" with a brief reason
- Example: "Based on your focus on fitness this week -- Episode 47: The 5am Brotherhood Routine"

#### 3. Admin-Triggered Recommendations

Keith can manually flag a product recommendation for a specific user from the admin dashboard. This overrides the algorithmic suggestions and appears at the next natural touchpoint.

### Revenue Tracking

Every recommendation includes a tracking ID:

```
{
  recommendation_id: "rec_abc123",
  user_id: "user_xyz",
  product: "man_in_the_arena_tour",
  trigger_signal: "low_partner_dial_communication",
  presented_at: "2026-03-24T14:30:00Z",
  placement: "end_of_conversation",
  outcome: "clicked" | "dismissed" | "purchased" | "expired"
}
```

Track the full funnel: recommendation shown --> clicked --> external page visited --> purchased (via UTM parameters or Stripe metadata). Report conversion rates per product, per trigger signal, and per placement.

## Implementation

```
src/services/recommendation.service.ts    -- rule engine and product matching
src/services/product-catalog.service.ts    -- product definitions and availability
src/models/recommendation.model.ts         -- recommendation and tracking types
```

### RecommendationService API

- `getRecommendations(userId) -> Recommendation[]` -- returns top 3 ranked suggestions
- `recordImpression(recommendationId) -> void` -- tracks that user saw the recommendation
- `recordOutcome(recommendationId, outcome) -> void` -- tracks click/dismiss/purchase
- `dismissRecommendation(userId, productId) -> void` -- starts cooldown timer

### Integration with Coaching AI

When generating a coaching response, the conversation service:
1. Calls `getRecommendations(userId)`
2. If a recommendation matches the current conversation topic, appends context to the system prompt
3. The AI decides whether and how to weave it into the response naturally
4. If the AI includes the recommendation, `recordImpression` is called

The AI is instructed to NEVER be pushy. If the conversation is emotionally heavy or crisis-related, recommendations are suppressed entirely.

## Consequences

### Positive

- Surfaces Keith's products at moments when users are most likely to benefit from them
- Recommendations feel like coaching advice, not advertising, because they come through Keith's AI voice
- Revenue tracking provides clear ROI data on which products and which trigger signals convert best
- Cooldown system prevents recommendation fatigue
- Keith can manually recommend products to specific users

### Negative

- Risk of eroding trust if recommendations feel salesy despite careful tone management
- Rule-based system may miss good recommendation opportunities (no collaborative filtering)
- Conversion tracking across external purchase flows (Stripe, event ticketing) requires UTM discipline
- Additional complexity in the coaching prompt (recommendation context injection)

### Mitigations

- A/B test recommendation frequency: start conservative (max 1 per week per user) and increase if feedback is positive
- User feedback signal: if a user gives thumbs-down on a message containing a recommendation, reduce future recommendation frequency for that user
- Quarterly review of conversion data to prune low-performing recommendation rules
- Never recommend during crisis conversations or when SafetyGuardrails has flagged the session
